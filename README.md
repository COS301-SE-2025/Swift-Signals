# Team InsideInsights - Swift-Signals

Swift Signals is a data-driven, simulation-powered traffic light optimization platform designed to address traffic congestion in urban environments. Developed in collaboration with Southern Cross Solutions, the project aims to equip municipal traffic departments with advanced tools to analyse intersection performance and improve traffic signal efficiency using machine learning. Traffic congestion remains one of South Africa‚Äôs most costly infrastructure challenges, with the South African Road Federation estimating annual productivity losses of approximately R1 billion. Swift Signals tackles this issue by providing a modular, web-based platform capable of simulating real-world intersection behaviour and dynamically optimizing traffic light phase configurations using historical traffic data. The system leverages modern software engineering principles‚Äîmicroservices, containerization, and continuous deployment pipelines‚Äîto deliver a scalable and maintainable solution.

## Git Workflow

This project follows a structured Git workflow to ensure code quality and maintainability. The branching strategy is based on a hierarchy of branches, with `main` representing production-ready code and `dev` as the integration branch for new features.

### Branching Strategy

- **`main`**: Production-ready code.
- **`dev`**: Development and staging.
- **`feature/microservice/<feature_name>`**: For new feature development.
- **`hotfix/<critical-security-patch_name>`**: For urgent fixes on `main`.

For a detailed guide on our Git workflow, including commands and best practices, please see the [**Git Workflow Setup Guide**](scripts/README.md).

## Getting Started

To get a local copy up and running, follow these simple steps.

### Prerequisites

- Docker and Docker Compose
- Git

### Installation

1. Clone the repo
   ```sh
   git clone https://github.com/COS301-SE-2025/Swift-Signals.git
   ```
2. Navigate to the project directory
   ```sh
   cd Swift-Signals
   ```

## Running the Application

1. Navigate to the development deployment directory:
    ```sh
    cd deployments/development
    ```
2. Start the application using Docker Compose:
    ```sh
    docker-compose up -d
    ```
3. The frontend will be available at `http://localhost:80`.

## Testing

### Go Services

To run the tests for the Go services, navigate to the service directory and use the `make` command.

- **user-service:**
  ```sh
  cd user-service
  make test
  ```
- **intersection-service:**
  ```sh
  cd intersection-service
  make unit-test
  ```

### Python Services

To run the tests for the Python services, navigate to the service directory and use `pytest`.

- **simulation-service & optimization-service:**
  ```sh
  cd <service-directory>
  pytest
  ```

### Frontend

There is currently no test script configured for the frontend.
